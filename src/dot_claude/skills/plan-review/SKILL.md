# /plan-review - 計画レビューループ

plan-reviewer エージェント（読み取り専用・1 起動 = 1 ラウンド）を使い、計画が合格するまでレビューループを回す。修正と `## レビュー履歴` の記録はこのスキルを実行するあなた（呼び出し元）の責務。

## 手順

1. **対象の特定**: 引数でパスが渡されていればそれを対象にする。無ければ現在作成中の計画をファイルに保存する（plan mode の plan ファイル、それ以外は `<project_root>/.ignore/ai/plan.md`）。
2. **レビュー起動**: Agent tool で `plan-reviewer` を起動し、plan ファイルの**パスのみ**を渡す（必要なら背景説明を 1〜2 文添えてよい。計画本文のインライン貼り付けは禁止）。判定（合格 / 要修正）と、確度（CONFIRMED / PLAUSIBLE）付きの指摘リストが返る。
3. **指摘の取捨**: plan-reviewer は会話コンテキストを持たないため、的外れな指摘があり得る。各指摘を会話の文脈と照らして妥当なものだけ採用する。PLAUSIBLE の指摘は前提が実際に成り立つかを特に確認する。
4. **修正の適用**: 採用した指摘を plan ファイルに反映する。Critical/Major は常に、修正コスト低の Minor も修正。修正コスト高の Minor は残してよい。
5. **履歴の記録**: plan ファイル末尾の `## レビュー履歴` セクションに `### ラウンド N` を追記する。内容: そのラウンドの指摘（棄却したものは棄却理由付き）と適用した修正。**履歴 1 ラウンド = plan-reviewer 起動 1 回**を厳守する — 履歴とエージェント起動記録の突き合わせでレビュー実施を検証できる状態を保つため、起動していないラウンドを書かない。
6. **反復**: 判定が「合格」になるまで 2〜5 を繰り返す。上限 4 ラウンド。収束しなければ最良版を提示し、未解決の指摘を明記する。

## レビュー履歴フォーマット

````
## レビュー履歴

### ラウンド 1
- **指摘**: [Critical/コスト低/CONFIRMED] 指摘内容
  - 対応: どう修正したか
- **指摘**: [Minor/コスト低/PLAUSIBLE] 指摘内容
  - 対応: 棄却 — 棄却理由（会話文脈上の根拠）

### ラウンド 2
- 指摘なし（合格）
````

## 完了報告

ユーザーへの報告に以下を含める: 実施ラウンド数、最終判定、棄却した指摘とその理由の要約。

## スキップ条件

- ユーザーが明示的に「レビュー不要」と言った場合
- 既にこのループで合格済みの計画を再提示するだけの場合
